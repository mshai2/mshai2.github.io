by Maxim Shainski

ICS4U

Hi, my name is Maxim Shainski and this is my final project for the course ICS4U. In my portfolio, you can find my top 10 modules that I worked on throughout this class. These modules come from Codecademy as well as Khan Academy. I also decided to include some of my projects and the creative coding challenge that we had during this class.

  • Github

Top 10 modules

Advanced Javascript: Natural Simulations

This module was important as it went deep into mathematical concepts as well as simulations of objects. It used simulated real forces, such as gravity, and important concepts, such as vectors. Previously to this module, computer science and math were loosely related to me. However, after this module, it's been made very evident how interconnected the two fields are. I'm currently hoping to pursue a career in software engineering, so I'll apply the mathematical concepts I've learned to my future learning.

  • Projects

Algorithms

This module is what made me appreciate effiicency. By introducing many different ways of sorting, it made obvious how every problem has many different solutions, and your job is not just to solve the problem, but figure out the most efficient way of doing so. However, there's the other side of the coin, where it's very difficiult to be 100 percent efficient, and many times impossible. This is where we employ heuristics. As I continue my coding journey, I will most likely impliment many of these sorting algorithms in my future code.

  • Projects

Learn Git & GitHub: Git Teamwork

Being able to properly use GitHub is a very important skill, as it's widely used in the programming field. This module taught how to navigate through files, make new repositories, add them to staging areas and commit them to the main. From my understanding, these are the basic essentials for GitHub, which made me far more confident in my ability to use it. In the future, I hope to have a career involving software engineering, so the skills I learned from this module will be used when I collaborate with my future workmates.

  • Project

Create a Back-End App with JavaScript: JavaScript Objects, Modules, and Browser Compatibility

Of this entire skill path, this module was the most educative and exciting for me. Objects were something I very loosely understood, same goes for their applications. However, this module taught principles of objects and methods, as well as getters and setters and other object related concepts. It showed the efficiency of objects. However, it also showed how to use node modules. These modules made the code a lot more concise and made manipulating them with other modules very efficient in terms of lines of code. I'll be incorporating far more objects and modules in my future code to further test their applications as opposed to function oriented programming.

  • Project

Create a Back-End App with JavaScript: SQL for Back-End Development

I'm personally very interested in penetration testing and cyber security. One of the most basic ways of hacking a website is through an SQLInjection. I hadn't really understood how it works until now. This module shows how to use SQL to make a back-end. Back-end is something I always struggled with, so this module helped expand on my knowledge of it. In the future, I'd like to make a website using SQL and penetration test my own website.

  • More

Create a Front-End App with React: JavaScript Iterators, Objects, and Classes

Iterators and classes go hand in hand with objects. They make objects far more powerful and versatile. Specficially, learning about classes with their instantiation abilities with objects completely changed how I'd code future games with multiple objects. Whenever I want to make a game that has several objects in the future, I now have the very powerful tool of classes to make it easier.

  • More

Create a Front-End App with React: Introduction to React

React is a JavaScript library designed for building user interfaces. This module teaches the syntax, which is very similar to HTML, as well as it's applications. It was important, as using libraries is an essential part of being a programmer. So, I've learned how to learn new libraries. I'll use react if I make something like an app in the future.
